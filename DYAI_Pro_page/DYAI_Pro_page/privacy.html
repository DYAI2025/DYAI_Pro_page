9. Ergänzende Hinweise für InstaFile (Chrome Extension)

InstaFile ist eine von DYAI entwickelte Chrome Extension, die markierten Text als lokale Datei speichert. Die Erweiterung verarbeitet ausschließlich lokal im Browser und sendet keine Daten an Server.

9.1 Datenverarbeitung bei InstaFile

  • Keine Erhebung personenbezogener Daten: InstaFile erfasst, speichert oder überträgt keine personenbezogenen Daten (keine Namen, E-Mail-Adressen, IP-Adressen oder Nutzungsstatistiken).
  • Lokale Verarbeitung: Markierte Inhalte werden ausschließlich zum Zeitpunkt des Speichervorgangs verarbeitet, um die gewünschte Datei zu erstellen. Es erfolgt keine dauerhafte Speicherung.
  • Einstellungen: Die Extension speichert nur lokale Konfigurationseinstellungen (z. B. Namensmuster für Dateien, UI-Präferenzen) im Browser-Storage (chrome.storage.local). Diese Daten verlassen niemals Ihren Browser.
  • Keine Tracker oder Analytik: InstaFile verwendet keine Analyse-, Tracking- oder Werbeskripte. Es werden keine Nutzungsdaten erhoben oder ausgewertet.
  • Kein Remote-Code: Der gesamte Code ist im Extension-Paket enthalten. Es werden keine Skripte von externen Servern nachgeladen.

9.2 Berechtigungen (Chrome Permissions)

InstaFile benötigt folgende Browser-Berechtigungen:

  • downloads: Zum Erstellen und Speichern von Dateien im Download-Ordner.
  • storage: Zum Speichern lokaler Einstellungen (Namensmuster, UI-Konfiguration).
  • contextMenus: Zum Bereitstellen des Kontextmenü-Eintrags „Auswahl als Datei speichern".
  • scripting/activeTab/tabs: Zum Lesen der aktuellen Textauswahl und des Seitentitels (zur Bildung sprechender Dateinamen).
  • notifications (optional): Für optionale System-Benachrichtigungen („Datei gespeichert").

Diese Berechtigungen dienen ausschließlich der Funktionalität der Extension. Es werden keine Daten an DYAI oder Dritte übermittelt.

9.3 DSGVO-Konformität

InstaFile ist Privacy-by-Design und erfüllt die Anforderungen der DSGVO:

  • Keine Verarbeitung personenbezogener Daten
  • Keine Weitergabe an Dritte
  • Vollständige Kontrolle durch den Nutzer
  • Transparenz durch Open-Source-Code (einsehbar auf GitHub)

9.4 Open Source

Der Quellcode von InstaFile ist öffentlich verfügbar. Sie können ihn jederzeit einsehen, um sich von der Datenschutzkonformität zu überzeugen.
